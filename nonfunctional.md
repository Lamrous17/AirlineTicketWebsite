Нефункциональные требования (NFR)

NFR-1 Система должна быть реализована на языке Python версии не ниже 3.8.

NFR-2 Система должна использовать фреймворк Django версии не ниже 3.2.

NFR-3 Система должна использовать СУБД PostgreSQL версии не ниже 12.

NFR-4 Основным языком пользовательского интерфейса должен быть русский язык.

NFR-5 Время поиска рейсов не должно превышать 5 секунд при стандартной нагрузке.

NFR-6 Время аутентификации и авторизации пользователя не должно превышать 3 секунд.

NFR-7 Система должна выполнять проверку корректности вводимых пользователем данных.

NFR-8 Система должна обеспечивать защиту персональных данных в соответствии с требованиями Федерального закона №152-ФЗ «О персональных данных».

NFR-9 Система должна обеспечивать целостность данных при одновременной работе нескольких пользователей.

NFR-10 Интерфейс системы должен быть интуитивно понятным и выполненным в едином стиле.

NFR-11 Сообщения об ошибках должны содержать описание проблемы и рекомендации по её исправлению.

NFR-12 Система должна корректно работать в современных веб-браузерах.

NFR-13 Документация проекта должна быть оформлена в соответствии с требованиями ГОСТ 7.32-2017.
